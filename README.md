# 签字页批量生成系统

IPO 签字页批量生成工具，支持上传 DOCX 模板、创建项目、绑定模板、填写变量、批量生成下载。

---

## 本地启动步骤

### 1. 启动后端

```bash
cd backend
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

后端启动后访问 http://localhost:8000/api/health 确认运行正常。

### 2. 访问前端

后端启动后，在浏览器中访问：

```
http://localhost:8000/frontend/index.html
```

前端静态文件由后端在 `/frontend` 路径下提供服务，与 `/api` 路径不冲突。

## junhe/ 目录说明

`junhe/` 目录位于项目根目录（`junhe-signature/junhe/`），运行时由系统自动创建，**无需手动创建**，也不提交到 git。

目录结构：

```
junhe/
├── {templateId}/
│   └── xxx.docx          ← 模板源文件（上传时写入）
└── {projectId}/
    └── {templateId}/
        └── xxx.docx      ← 生成文件（生成时写入）
```

重新绑定模板时，会自动清空该项目下的所有生成文件。

## 模板变量格式

在 DOCX 模板文件中，使用 `{{变量名}}` 作为占位符：

- 变量名只允许英文字母、数字、下划线
- 系统自动解析模板中所有占位符，提取变量列表
- 多个模板绑定同一项目时，自动对变量做并集去重

常见变量示例：

| 变量名 key | 中文名 label | 示例值 |
|---|---|---|
| `law_firm_name` | 律师事务所名称 | 君合律师事务所 |
| `lawyer_head` | 事务所负责人 | 张三 |
| `company_name` | 发行人名称 | XX 股份有限公司 |
| `sign_year` | 签字年份 | 2026 |
| `sign_month` | 签字月份 | 06 |
| `sign_day` | 签字日期 | 27 |

## 使用流程

1. **上传模板**：进入「模板管理」→「上传模板」，上传 DOCX 文件，系统自动解析变量占位符。

2. **创建项目**：进入「项目管理」→「新建项目」，输入项目名称后提交。

3. **绑定模板**：在项目详情页，点击「重新绑定模板」，勾选需要的模板，点击「确认绑定」。系统自动对所有绑定模板的变量做去重合并。

4. **填写变量**：在「变量填写」区块中：
   - **页面填写**：直接在表格中填写每个变量的中文名（label）和填写值（value）
   - **Excel 导入**：下载 Excel 模板，填写后上传导入

5. **生成文件**：变量填写完成后，「生成与下载」区块显示，点击「生成签字页」批量生成所有 DOCX。

6. **下载文件**：单独下载每个生成文件，或点击「打包下载全部」获取 ZIP 压缩包。

## doc/ 目录说明

`doc/` 目录下的文件**仅供开发参考**，包含早期方案设计、接口示例及原型代码，不参与实际运行，也不应被引入生产环境。
