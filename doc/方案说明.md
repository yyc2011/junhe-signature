## 数据库表结构

### Template 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | String (UUID) | 主键 |
| name | String | 模板名称，去除后缀，全局唯一 |
| file_path | String | 源文件路径 `/junhe/{id}/xxx.docx` |
| variable_keys | JSON | `[{"key":"law_firm_name","label":"","type":"text"}]` |
| created_at | DateTime | 创建时间 |
| updated_at | DateTime | 更新时间（覆盖上传时更新） |

> `variable_keys` 的 label 在上传时由系统自动解析 key，label 初始为空字符串，在项目绑定模板后由用户在项目维度填写。

### Project 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | String (UUID) | 主键 |
| name | String | 项目名称，用户手动输入 |
| status | Enum | `draft` / `filling` / `generated` |
| created_at | DateTime | 创建时间 |

### ProjectTemplate 表（关联表）

| 字段 | 类型 | 说明 |
|------|------|------|
| project_id | String (FK) | 关联 Project |
| template_id | String (FK) | 关联 Template |

> 联合主键 (project_id, template_id)

### ProjectVariable 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | String (UUID) | 主键 |
| project_id | String (FK) | 关联 Project |
| key | String | 变量名，如 `law_firm_name` |
| label | String | 中文显示名，如 `律师事务所名称` |
| value | String | 用户填写的值，默认空 |
| type | Enum | `text` / `date`，默认 `text` |

> (project_id, key) 联合唯一索引，保证同一项目内变量不重复

### GeneratedFile 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | String (UUID) | 主键 |
| project_id | String (FK) | 关联 Project |
| template_id | String (FK) | 关联 Template |
| file_path | String | 生成文件路径 |
| created_at | DateTime | 生成时间 |

## 接口设计

### 模板相关

| Method | Path | 说明 |
|--------|------|------|
| GET | /api/templates | 获取模板列表 |
| POST | /api/templates/upload | 上传模板 DOCX（multipart） |

### 项目相关

| Method | Path | 说明 |
|--------|------|------|
| GET | /api/projects | 获取项目列表 |
| POST | /api/projects | 创建项目 |
| GET | /api/projects/{id} | 获取项目详情（含绑定模板和变量） |
| POST | /api/projects/{id}/bind | 绑定模板（清空旧绑定和文件，返回去重变量列表和 Excel） |
| GET | /api/projects/{id}/variables/excel | 下载变量 Excel 模板 |
| POST | /api/projects/{id}/variables | 提交变量值（JSON 或 Excel 上传） |
| POST | /api/projects/{id}/generate | 生成所有 DOCX 文件 |
| GET | /api/projects/{id}/files/{template_id}/download | 下载单个生成文件 |
| GET | /api/projects/{id}/download | 打包下载所有生成文件 ZIP |

## 业务流程

```
1. 上传模板
   └── 解析变量 key → 存 Template 表 → 保存 DOCX 到 /junhe/{templateId}/

2. 创建项目
   └── 输入 name → 存 Project 表（status=draft）

3. 绑定模板（可重复操作，每次清空重来）
   └── 清空 ProjectTemplate / ProjectVariable / GeneratedFile / 磁盘生成文件
   └── 写入 ProjectTemplate
   └── 对所有模板的 variable_keys 做并集去重
   └── 写入 ProjectVariable（key+label空+value空）
   └── 返回变量列表 + Excel 下载链接
   └── Project.status → filling

4. 填写变量（二选一）
   ├── 页面表单：列出所有变量，填写 label + value，提交
   └── Excel：下载模板 → 填写 → 上传

5. 生成文件
   └── 检查所有变量 label + value 非空
   └── 遍历绑定模板，docxtpl 渲染，保存到 /junhe/{projectId}/{templateId}/
   └── 写入 GeneratedFile 表
   └── Project.status → generated

6. 下载
   ├── 单个文件下载
   └── 全部打包 ZIP 下载
```

## 变量约定

模板 DOCX 文件内占位符格式统一使用 `{{变量名}}`，变量名只允许英文字母、数字、下划线。

常见变量 key 参考（实际以模板解析为准）：

```
law_firm_name     律师事务所名称
lawyer_head       事务所负责人
lawyer_1          经办律师1
lawyer_2          经办律师2
company_name      发行人名称
shareholder_name  股东姓名
sign_year         签字年份
sign_month        签字月份
sign_day          签字日期
```

Excel 变量文件格式（两列）：

| key | label | value |
|-----|-------|-------|
| law_firm_name | 律师事务所名称 | 君合律师事务所 |
| lawyer_head | 事务所负责人 | 张三 |

## 文件存储约定

```
/junhe/{templateId}/{name}.docx          # 模板源文件，上传时覆盖
/junhe/{projectId}/{templateId}/{name}.docx  # 生成文件，生成时覆盖
```

重新绑定模板时，删除 `/junhe/{projectId}/` 下所有内容。
